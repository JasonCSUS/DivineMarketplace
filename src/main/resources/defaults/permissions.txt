DivineMarketplace permissions reference
=======================================

This file is copied to plugins/DivineMarketplace/permissions.txt on first startup.
It is a server-owner reference for LuckPerms or another permission manager; it is
not read back by the plugin at runtime.

Built-in player permissions
---------------------------
divinemarketplace.use
  Basic player access to /market and the marketplace GUI.

divinemarketplace.claim
  Allows opening claim views and claiming pending item/money claims.

Built-in admin permissions
--------------------------
divinemarketplace.admin
  Parent permission for full marketplace administration. The plugin.yml parent
  grants the built-in admin child permissions below.

divinemarketplace.admin.reload
  Reload config files, category data, custom definitions, and menu visuals.

divinemarketplace.admin.recalc
  Recalculate one market profile or all market profiles.

divinemarketplace.admin.price.set
  Manually set the current recommended price for a market entry.

divinemarketplace.admin.listing.cancel
  Cancel another player's listing from admin controls.

divinemarketplace.admin.history.view
  View admin history in-game.

divinemarketplace.admin.history.export
  Export admin history into readable text files.

divinemarketplace.admin.item.sort
  Move a market entry/category placement through admin sort commands.

divinemarketplace.admin.item.define
  Define/update held custom item metadata.

divinemarketplace.admin.enchant.define
  Define/update custom enchant metadata.

divinemarketplace.admin.flagged.view
  View flagged/review-needed custom item entries.

divinemarketplace.admin.collisions.view
  View custom item collision log entries in-game.

divinemarketplace.admin.inspect
  Inspect held item metadata, rule matches, and raw snapshot output.

divinemarketplace.admin.custom.override
  Mark held items as treat-as-vanilla or remove those overrides.

Config-defined listing tier permissions
---------------------------------------
Premium listing tier permissions are not hard-coded. They are read from
config.yml under listingPolicies.tiers. The bundled default config currently
ships these examples:

- divinemarketplace.tier.1
- divinemarketplace.tier.2
- divinemarketplace.tier.3

The plugin uses the first/highest matching configured tier to decide:
- max active listings
- listing duration

If you change listingPolicies.tiers in config.yml, mirror those permission nodes
in your LuckPerms setup and update this reference file for your staff.

Maintenance reminder
--------------------
When a new hard-coded built-in permission is added, update:
- src/main/resources/plugin.yml
- src/main/resources/defaults/permissions.txt
